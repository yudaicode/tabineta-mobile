# Android セットアップ：今やるべき？後でもいい？

「どのみち最終的には必要なんでしょ？」という疑問にお答えします。

---

## 📊 結論：開発フェーズによる

### 開発段階（今） → **不要** ✅

**簡易セットアップ（Android SDK なし）で十分です:**
- Web クライアント ID のみで OAuth が動作
- iOS/Android 両方でテスト可能
- Expo Go または Web ブラウザで開発

**必要になるのは:**
- ❌ Google Play にリリースする時
- ❌ ネイティブモジュールを追加する時
- ❌ カスタムネイティブコードを書く時

### 本番リリース時（後） → **必要** ⚠️

Google Play にアプリをリリースする際には必須:
- ✅ Android Studio
- ✅ リリース用キーストア（デバッグ用とは別）
- ✅ リリース用 SHA-1
- ✅ Android クライアント ID（リリース用 SHA-1 で作成）

## 🔑 重要な事実

### デバッグ用 SHA-1 ≠ リリース用 SHA-1

**デバッグ用（開発環境）:**
```
~/.android/debug.keystore から生成
開発・テスト用
自動生成される
```

**リリース用（本番環境）:**
```
自分で作成したキーストアから生成
Google Play リリース用
厳重に管理する必要がある
```

### つまり...

**今 Android クライアント ID を作成しても、リリース時に追加作業が必要です:**

1. 今作成する: デバッグ用 SHA-1 で Android クライアント ID
2. リリース時: リリース用 SHA-1 を**同じ Android クライアント ID に追加**

→ **デバッグ用とリリース用の SHA-1 を両方登録することになります**
